Learning Management System

The Exam Registration System (ERS) was developed to streamline the process of registering students for French exams offered by various countries and institutions. Designed for the University of Buea, the system simplifies registration, payment verification, and administrative oversight. It was built to address the challenges of manual registration and improve operational efficiency for both students and administrators.

Objectives
  • 1. Provide an easy-to-use platform for students to register for exams online.
  • 2. Ensure secure payment verification through receipt uploads.
  • 3. Equip administrators with tools to manage exam schedules and student data effectively.
  • 4. Create a scalable, secure, and efficient system using modern technologies.

Development Process
1. Requirement Gathering
  • Our initial step was to collaborate closely with the dean and the head of the electrical engineering department, who were leading the development of the software. We provided a feature list proposal and conducted meetings with the clients to validate whether these features aligned with their expectations and needs.
2. System Architecture
  • We proceeded to design a modular architecture, utilizing React for the frontend and Laravel for the backend, ensuring scalability and ease of maintenance.
  • For the database, we selected PostgreSQL, hosted securely on AWS to ensure optimal performance and reliability.
  • The application was containerized with Docker, and automated deployments were set up through GitHub CI/CD pipelines, streamlining the release process.
  • After finalizing the cost estimates for hosting and messaging services, we began designing wireframes, which were then presented for client approval before proceeding further.
3. Design and Prototyping
  • Once we received client approval for the wireframes, our UX designer began creating high-fidelity designs and prototypes in Figma.
  • After each design iteration, my team and I reviewed the designs to ensure accuracy, functionality, and adherence to project goals. We provided feedback to the designer, and once the core pages were finalized, we presented them to the client for their review.
  • Following the client's feedback and a few final adjustments, we walked the client through the entire design, demonstrating how the user experience had been integrated. Any further modifications were quickly implemented before moving into full-scale development.
4. Development
  • The frontend was developed using Next.js and Tailwind CSS. I opted for these technologies due to their performance optimizations, scalability, and ability to deliver faster load times, which are crucial for a seamless user experience.
  • We built a robust and dynamic system capable of handling up to 10,000 concurrent users. Ensuring security was a priority, and we implemented strong protection for sensitive user data, while also enabling the admin to efficiently manage and access the necessary information.
  • We integrated third-party services such as SendGrid for email notifications and Twilio for SMS alerts to ensure seamless communication with users throughout the registration process.
  • Responsive design was a key focus to ensure the platform was fully optimized for all devices, providing a smooth experience for both students and administrators.
  • Collaboration was essential, so we used Jira for task tracking, Slack and WhatsApp for team communication, and Git for source code management to maintain a streamlined development workflow.
5. Testing
  • We conducted unit testing on individual components to ensure that each functioned correctly in isolation.
  • Integration testing was performed across modules to verify that the system components worked seamlessly together.
  • To simulate peak registration periods, we performed load testing to assess the systemâ€™s performance under heavy traffic.
  • During a dedicated three-week QA cycle, we fixed any bugs that were identified and made performance improvements to ensure a stable and efficient system.
6. Deployment
  • We deployed the platform on AWS with NGINX to ensure reliable performance and scalability in the cloud environment.
  • Our builds and deployments were fully automated using GitHub Actions, ensuring a streamlined and efficient release process with minimal downtime.
