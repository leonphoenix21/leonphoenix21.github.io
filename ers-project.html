Exam Registration System

The Exam Registration System (ERS) was developed to streamline the process of registering students for French exams offered by various countries and institutions. Designed for the University of Buea, the system simplifies registration, payment verification, and administrative oversight. It was built to address the challenges of manual registration and improve operational efficiency for both students and administrators.

Objectives
  • 1. Provide an easy-to-use platform for students to register for exams online.
  • 2. Ensure secure payment verification through receipt uploads.
  • 3. Equip administrators with tools to manage exam schedules and student data effectively.
  • 4. Create a scalable, secure, and efficient system using modern technologies.
My Role

As the Project Manager and Frontend Architect, I led a team of five:

  • 2 Backend Developers

Responsibilities
  • Defined the project scope, timelines, and deliverables.

Development Process

  • Our initial step was to collaborate closely with the dean and the head of the electrical engineering department, who were leading the development of the software. We provided a feature list proposal and conducted meetings with the clients to validate whether these features aligned with their expectations and needs.
  • We proceeded to design a modular architecture, utilizing React for the frontend and Laravel for the backend, ensuring scalability and ease of maintenance.
  • For the database, we selected PostgreSQL, hosted securely on AWS to ensure optimal performance and reliability.
  • The application was containerized with Docker, and automated deployments were set up through GitHub CI/CD pipelines, streamlining the release process.
  • After finalizing the cost estimates for hosting and messaging services, we began designing wireframes, which were then presented for client approval before proceeding further.
  • Once we received client approval for the wireframes, our UX designer began creating high-fidelity designs and prototypes in Figma.
  • After each design iteration, my team and I reviewed the designs to ensure accuracy, functionality, and adherence to project goals.
  • Following the client's feedback and a few final adjustments, we walked the client through the entire design, demonstrating how the user experience had been integrated.
  • The frontend was developed using Next.js and Tailwind CSS, chosen for their performance optimizations and scalability.
  • We built a robust and dynamic system capable of handling up to 10,000 concurrent users, with strong protection for sensitive data.
  • Third-party services like SendGrid and Twilio were integrated to provide email notifications and SMS alerts.
  • Collaboration was paramount; we employed Jira, Slack, WhatsApp, and Git to maintain efficient teamwork and source code management.
  • Unit testing on individual components ensured each piece functioned correctly in isolation.
  • Integration testing verified seamless interaction among modules.
  • Load testing simulated peak usage periods to confirm stable performance under high traffic.
  • A thorough QA cycle identified and resolved any bugs, resulting in a more efficient system.
  • We deployed on AWS with NGINX, ensuring reliable performance and scalability in the cloud.
  • Our CI/CD pipelines were streamlined with GitHub Actions, minimizing downtime and automating releases.

Key Features

  • 1. Exam Registration: Students can browse a list of available exams, view prerequisites, and register for exams directly from their dashboard.
  • 2. Payment Verification: Upload payment receipts for exam registrations, which will be verified by administrators.
  • 3. Dashboard Access: View upcoming exams, schedules, and exam registration statuses. Receive automated notifications and reminders about exam deadlines and schedules.
  • 4. Profile Management: Update personal information, such as email addresses and phone numbers.
  • 5. Notifications: Receive SMS and email alerts about successful registration, payment verification, and reminders for upcoming exams.
  • 1. User Management: View, edit, and manage student accounts, including resetting passwords and updating profiles.
  • 2. Exam Schedule Management: Create, update, and delete exam schedules with detailed information, such as location, date, and time.
  • 3. Payment Verification: Review and verify uploaded payment receipts from students and mark them as verified. Send confirmation notifications upon successful payment verification.
  • 4. Bulk Notifications: Send announcements or updates to students via email or SMS regarding registration deadlines, exam schedules, or urgent updates.
  • 5. Reporting: Generate detailed reports on exam registrations, payments, and attendance. Export reports in PDF or Excel formats for record-keeping.
  • 1. Secure Authentication: Provide role-based access control, ensuring students and administrators access only their respective features.
  • 2. Responsive User Interface: Accessible on desktops, tablets, and smartphones with consistent performance.
  • 3. Notifications and Alerts: Real-time alerts ensure all users stay informed about updates, deadlines, and system activities.
  • 4. Data Security: Ensure all communications are secured using HTTPS, and sensitive data (like payment receipts) is encrypted at rest.

Challenges and Solutions

  • Challenge: Ensuring the system could handle peak registration traffic. Optimized database queries and implemented load balancing on AWS.
  • Challenge: Aligning the system with client requirements while managing a tight timeline. Maintained close communication with the client and conducted frequent review sessions.
  • Challenge: Securely managing sensitive student data. Enforced HTTPS, encrypted data at rest and in transit, and implemented role-based access control (RBAC).
